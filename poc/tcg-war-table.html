D07 — Git Integration

Feature Sprint 3 Rare

Overview

Parse git log and branch state to surface project context in the Mission Control dashboard. This powers the stats bar commit sparkline and the branch indicator.

Requirements

  • Extract commit history with branch tracking metadata
  • Parse current branch, ahead/behind counts
  • Surface recent commits in the stats bar
  • Provide commit-to-deliverable linking via conventional commit parsing

Technical Approach

Extend the existing gitParser service with a new getBranchMap() function that maps commit SHAs to branch names. The function will shell out to git for-each-ref and parse the output into a Map<string, string[]>.

"Every commit tells a story. This deliverable makes that story visible."

Acceptance Criteria

  • Stats bar shows commit count for last 7 days
  • Branch name displayed in command bar
  • Commits linked to deliverable IDs when present in message
  • No performance regression on dashboard load
