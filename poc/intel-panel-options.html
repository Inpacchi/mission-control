Specification

Integrate git context into the deliverable tracking pipeline so that each kanban card surfaces its associated commits, branch, and change summary.

Goals

  • Parse git log output linked to each deliverable ID
  • Expose branch name and last-commit date in the card stats zone
  • Keep file-watcher latency under 150ms

Implementation Notes

// gitParser.ts — entry point
export async function getDeliverableCommits(
  projectPath: string,
  deliverableId: string
): Promise<GitCommit[]> {
  const pattern = `(${deliverableId})`
  return parseGitLog(projectPath, pattern)
}
Assumption: commit messages follow the convention type(DNN): description. Deliverables without matching commits surface an empty history without error.
